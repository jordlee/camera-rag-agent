The value to the left in parentheses is the maximum number of characters including null terminators.And the right value of the colon is the minimum and maximum number of bytes for each member. If omitted, nullptr is fine. For example, when setting one character of 'A‘, the first 2 bytes + 'A‘ + null-terminator, totaling 4 bytes, should be prepared.

.. _cr-ftp-job-info:

CrFTPJobInfo
------------

Class describing FTP Job info.

Class for checking information on FTP Jobs registered using ControlFTPJobList(), which can be obtained with GetFTPJobList().

Further FTP Jobs can be controlled according to the status of FTP Jobs registered with ControlFTPJobList(). For example, you can suspension of content transfer in transit by specifying :ref:`CrFTPJobControlType_Suspend<cr-ftp-job-control-type>` to :ref:`ControlFTPJobList()<api-control-ftp-job-list>`.
See “:ref:`Control the FTP Jobs<op-control-the-ftp-jobs>`” for instructions on how to control FTP Jobs.

Member Variables

.. list-table::
  :widths: auto
  :header-rows: 1

  * - Name
    - Type
    - Summary
  
  * - serverId
    - CrInt32u
    - Server ID
  * - jobId
    - CrInt32u
    - Job ID
  * - slotId
    - CrFTPJobSlotId
    - Media SLOT number
  * - jobStatus
    - CrFTPJobStatus
    - Job Status
  * - chunkNum
    - CrInt32u
    - | Number of chunks
      | When transfer is complete : Total chunks
      | Transfer in process : Transfer complete chunks
  * - fileSize
    - CrInt64u
    - File size
  * - transferSize
    - CrInt64u
    - Transferred size
  * - clipName
    - CrInt8u*
    - Destination file name (\*1)
  * - mainName
    - CrInt8u*
    - Source file name (\*1)
  * - metaName
    - CrInt8u*
    - Meta file name of the Source file (\*1)

\*1 : The first 2 bytes are the number of characters including the null terminator. Basically fold back the file name set in CrFTPJobSetting class.
