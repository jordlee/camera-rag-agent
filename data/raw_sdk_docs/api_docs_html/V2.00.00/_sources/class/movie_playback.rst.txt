CrMoviePlaybackSetting
----------------------

Class describing Movie Playback setting information.

Member Variables

.. list-table::
    :widths: auto
    :header-rows: 1

    * - Name
      - Type ([#f1]_)
      - Summary

    * - slotId
      - CrSlotNumber
      - Slot number of media include target content.
    * - contentsId
      - CrInt32u
      - ContentsId that get by :ref:`CrContentsInfo<cr-contents-info>`.
    * - fileId
      - CrInt32u
      - FileId that get by :ref:`CrContentsFile<cr-contents-file>`.
    * - ipAddress
      - CrInt8u*  (16:10-18)
      - IP address of host device.
    * - downTime
      - CrInt32u
      - | Time to check for no delivery data received.
        | Unit is milliseconds.
        | When 0 is specified, no delivery check is performed.
    * - videoPort
      - CrInt32u
      - | Receive port number for image data.
        | Specify a number between 0 or 49152~65535(Range of ports that users can use for socket communication).
        | 0 means default value specification. In that case, it will be replaced by 55001.
        | Note:
        | 55003 and 55005 are unavailable due to reserved ports.
    * - audioPort
      - CrInt32u
      - | Receive port number for audio data.
        | Specify a number between 0 or 49152~65535(Range of ports that users can use for socket communication).
        | 0 means default value specification. In that case, it will be replaced by 55003.
        | Note:
        | 55001 and 55005 are unavailable due to reserved ports.
    * - metaPort
      - CrInt32u
      - | Receive port number for meta data at received image data.
        | Specify a number between 0 or 49152~65535(Range of ports that users can use for socket communication).
        | 0 means default value specification. In that case, it will be replaced by 55005.
        | Note:
        | 55001 and 55003 are unavailable due to reserved ports.

.. rubric:: FootNotes

.. [#f1] String member. Must be set to the number of characters including the null-terminator in the first 2 bytes. The value to the left in parentheses is the maximum number of characters including null terminators.And the right value of the colon is the minimum and maximum number of bytes for each member. If omitted, nullptr is fine. For example, when setting one character of 'A‘, the first 2 bytes + 'A‘ + null-terminator, totaling 4 bytes, should be prepared.
